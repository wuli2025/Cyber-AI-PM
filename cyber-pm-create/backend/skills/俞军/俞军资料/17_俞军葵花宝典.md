# 俞军产品方法论葵花宝典

## 一、产品本质论

### 产品的定义
> 产品是一种解决用户问题的解决方案

### 产品价值
> 产品价值 = 用户价值 = 新体验 - 旧体验 - 替换成本

### 核心思维
1. 用户价值第一
2. 需求驱动而非技术驱动
3. 简单直接

---

## 二、需求分析论

### 需求三问
1. 用户是否真的需要？
2. 这个问题有多重要？
3. 我们能解决吗？

### 需求真伪判断
| 类型 | 特征 | 策略 |
|------|------|------|
| 真需求 | 用户愿意付费/改变行为 | 优先满足 |
| 伪需求 | 用户说不清/不会做 | 谨慎投入 |
| 弱需求 | 可有可无 | 看资源情况 |

### 需求分析框架
```
用户 → 场景 → 问题 → 需求 → 方案
```

---

## 三、用户价值论

### 用户价值公式
```
用户价值 = 新体验 - 旧体验 - 替换成本
```

### 公式解读
- **新体验**：使用新产品获得的价值
- **旧体验**：原有方案的价值
- **替换成本**：迁移到新方案的成本

### 提升用户价值
1. 提升新体验
2. 降低旧体验的参照感
3. 降低替换成本

---

## 四、决策方法论

### 决策框架
```
明确目标 → 列出选项 → 评估选项 → 做出选择 → 检验结果
```

### 决策原则
1. 用数据说话，但不迷信数据
2. 逻辑推理，假设验证
3. 权衡取舍，系统思考
4. 接受不确定性

### 常见决策陷阱
- 确认偏误
- 锚定效应
- 损失厌恶
- 过度自信

---

## 五、产品设计论

### 设计原则
1. **有用**：解决真实问题
2. **可用**：易于使用
3. **想用**：用户愿意用
4. **商业可行**：能产生价值

### 设计思考框架
```
Empathize（同理心）→ Define（定义问题）→ Ideate（构思方案）→ Prototype（原型）→ Test（测试）
```

### 极简设计
- 功能极简
- 流程极简
- 界面极简
- 概念极简

---

## 六、优先级论

### 优先级判断
1. **价值大小**：能创造多大价值
2. **紧急程度**：时间要求
3. **实现成本**：资源投入
4. **风险程度**：不确定性

### 优先级模型

#### 四象限
```
        | 紧急
   重要 | --------|--------
        |  救火  |  规划
--------|--------|--------
        |  说不  |   做
   不重要|--------|--------
              不紧急
```

#### RICE
```
R = 用户覆盖量 × 影响程度 × 置信度 / 投入精力
```

## 七、数据驱动论

### 数据思维
1. 定义指标
2. 收集数据
3. 分析数据
4. 验证假设
5. 指导决策

### 常见分析方法
- 对比分析
- 趋势分析
- 漏斗分析
- 留存分析
- 归因分析

### 数据陷阱
- 相关性≠因果性
- 样本偏差
- 辛普森悖论
- 数据选择偏差

---

## 八、迭代进化论

### 迭代思维
1. MVP原则
2. 快速验证
3. 持续优化
4. 小步快跑

### 迭代节奏
- 周迭代：修复优化
- 月迭代：小功能
- 季度迭代：大版本

### 复盘方法
```
回顾目标 → 对比结果 → 分析原因 → 总结经验 → 改进计划
```

## 九、团队协作论

### 角色定位
- 产品经理：用户代言人
- 开发：技术实现
- 设计：体验优化
- 测试：质量保障
- 运营：用户增长

### 沟通原则
1. 讲清楚"为什么做"
2. 倾听理解对方立场
3. 用数据说服
4. 尊重专业判断

### 跨部门协作
- 建立信任
- 明确目标
- 主动协调
- 及时同步

## 十一、常见问题论

### Q1: 需求太多怎么办？
A: 用优先级模型筛选，聚焦核心需求

### Q2: 开发说做不了怎么办？
A: 理解技术约束，探索替代方案

### Q3: 需求变更怎么办？
A: 评估影响，走变更流程

### Q4: 数据和直觉冲突怎么办？
A: 深挖数据，理解原因，不盲目决策

### Q5: 竞品抄我们怎么办？
A: 保持迭代速度，建立壁垒

## 十二、核心金句

1. 产品价值第一
2. 用户价值大于一切
3. 简单直接
4. 数据驱动但不迷信数据
5. 做减法比做加法难
6. 快速试错，持续迭代
7. 没有最优解，只有更优解
8. 理解用户，而非代表用户
9. 商业价值与用户价值平衡
10. 长期视角，不牺牲未来
