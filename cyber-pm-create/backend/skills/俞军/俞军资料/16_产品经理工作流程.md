### 4. 需求优先级

#### 常用方法

##### 四象限法则
```
        | 紧急
   重要 | --------|--------
        |  马上做  |  计划做
--------|--------|--------
        |  委托   |   忽略
   不重要|--------|--------
              不紧急
```

##### RICE模型
```
优先级分数 = (Reach × Impact × Confidence) / Effort
```

##### MoSCoW法则
- Must have（必须有）
- Should have（应该有）
- Could have（可以有）
- Won't have（这次不做）

## 九、标准化流程

### 日常流程

```
周一
├── 周会：同步进度
├── 问题协调
└── 本周计划

周二-周四
├── 需求分析
├── 产品设计
├── 需求评审
└── 日常沟通

周五
├── 周报
├── 下周计划
└── 复盘总结
```

---

### 项目流程

```
阶段      时间     产出物
─────────────────────────────
需求阶段   1-2周   需求文档、优先级
设计阶段   1-2周   原型、PRD
开发阶段   2-4周   代码、测试用例
测试阶段   1-2周   测试报告
上线阶段   1周     上线报告
运营阶段   持续    数据报告
```
