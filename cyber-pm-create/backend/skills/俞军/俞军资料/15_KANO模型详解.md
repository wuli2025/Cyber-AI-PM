# KANO模型详解

## 一、模型概述

### 起源
KANO模型由东京理工大学教授狩野纪昭(Noriaki Kano)于1984年提出，用于分析用户需求对满意度的影响。

### 核心价值
- 识别不同类型的需求
- 指导产品功能优先级
- 优化资源投入产出比

---

## 二、需求分类

### 1. 基本型需求（Must-be）

#### 特征
- 用户认为"必须有"的属性
- 不满足会强烈不满
- 满足后不会明显提升满意
- 属于"理所当然"的需求

#### 案例
| 产品类型 | 基本型需求 |
|----------|-----------|
| 电商APP | 支付安全 |
| 社交APP | 消息收发 |
| 搜索产品 | 搜索准确性 |
| 外卖APP | 下单功能 |

#### 管理策略
- 持续稳定提供
- 不需要过多投入优化
- 避免出现重大问题

---

### 2. 期望型需求（One-dimensional）

#### 特征
- 用户期望得到的需求
- 满足程度与满意度线性相关
- 做得越好越满意
- 是竞品之间比较的主要领域

#### 案例
| 产品类型 | 期望型需求 |
|----------|-----------|
| 电商APP | 价格优惠 |
| 社交APP | 好友互动 |
| 搜索产品 | 搜索速度 |
| 外卖APP | 配送速度 |

#### 管理策略
- 尽力满足
- 适度投入优化
- 与竞品比较差异化

---

### 3. 兴奋型需求（Attractive）

#### 特征
- 用户未曾期望的需求
- 满足后会大幅提升满意度
- 不满足也不会不满
- 往往是创新点所在

#### 案例
| 产品类型 | 兴奋型需求 |
|----------|-----------|
| 电商APP | AR试穿 |
| 社交APP | 趣味滤镜 |
| 搜索产品 | 智能推荐 |
| 外卖APP | 自动推荐 |

#### 管理策略
- 积极探索
- 寻找创新机会
- 创造差异化优势

---

### 4. 无差异需求（Indifferent）

#### 特征
- 用户不在意的需求
- 无论满不满意对满意度无影响
- 投入资源往往是浪费

#### 管理策略
- 避免投入
- 节省资源

---

### 5. 反向型需求（Reverse）

#### 特征
- 用户不期望的需求
- 做得越多满意度越低
- 可能触犯部分用户

#### 案例
- 过多广告推送
- 强制注册登录
- 复杂操作流程

#### 管理策略
- 避免提供
- 必要时降低

---

## 三、评估方法

### 问卷设计

#### 问题形式
针对每个需求设计两个问题：
1. **正向问题**：如果提供该功能，您的感受是？
2. **负向问题**：如果不提供该功能，您的感受是？

#### 选项设计
| 选项 | 正向问题 | 负向问题 |
|------|----------|----------|
| 很喜欢 | 1 | 5 |
| 理应如此 | 2 | 4 |
| 无所谓 | 3 | 3 |
| 勉强接受 | 4 | 2 |
| 很不喜欢 | 5 | 1 |

---

### 结果分析

#### Better-Worse系数

| 系数 | 计算公式 | 含义 |
|------|----------|------|
| Better | (A+O)/(A+O+M+I) | 增加后的满意度提升 |
| Worse | -(E+M)/(A+O+M+I) | 不提供时的满意度下降 |

#### 散点图分析
```
增加满意 ↑         降低满意 ↓
    |                |
    |   兴奋型需求    |
    |      A         |
----+------------------+---- 降低不满意
    |   基本型需求     | 基础不满意
    |      M         |
    |                |
    |   期望型需求    | 无差异需求
    |      O         |
    |                |
    |   反向型需求    |
    |      R         |
    +----------------+--->
  增加满意           增加不满意
```

## 四、应用场景

### 1. 需求优先级排序

#### 优先级参考
1. 基本型需求（必须满足）
2. 期望型需求（尽力满足）
3. 兴奋型需求（探索满足）
4. 无差异需求（避免投入）
5. 反向型需求（避免提供）

---

### 2. 产品规划

#### 产品迭代路径
```
第一阶段：完善基本型需求
    ↓
第二阶段：优化期望型需求
    ↓
第三阶段：探索兴奋型需求
```

---

### 3. 用户研究

#### 识别需求类型
- 通过调研区分不同类型需求
- 量化需求的重要程度
- 指导调研资源分配

---

## 五、实施步骤

### Step 1: 收集需求
- 用户访谈
- 竞品分析
- 数据分析
- 内部讨论

### Step 2: 设计问卷
- 针对每个需求设计正负向问题
- 控制问卷长度
- 选择目标用户

### Step 3: 执行调研
- 线上/线下问卷
- 保证样本量
- 数据清洗

### Step 4: 数据分析
- 计算各选项占比
- 绘制KANO分类图
- 计算Better-Worse系数

### Step 5: 结果应用
- 需求分类
- 优先级排序
- 产品规划建议

---

## 六、注意事项

### 局限性与注意事项

1. **动态变化**
   - 兴奋型需求可能变为期望型
   - 期望型可能变为基本型
   - 需定期重新评估

2. **文化差异**
   - 不同市场/用户群体有差异
   - 需要结合实际情况判断

3. **样本代表性**
   - 确保样本代表目标用户
   - 注意样本量

4. **结合其他方法**
   - KANO只是参考
   - 需结合业务判断
   - 考虑资源限制
