# PRD需求文档写作指南

## 一、PRD概述

### 什么是PRD
Product Requirement Document，产品需求文档

### PRD的作用
- 明确产品需求
- 统一团队理解
- 指导开发测试
- 记录决策过程

### PRD vs 其他文档

| 文档类型 | 关注点 | 受众 |
|----------|--------|------|
| PRD | 需求细节 | 开发、测试、设计 |
| BRD | 商业价值 | 管理层 |
| MRD |市场需求 | 市场、运营 |
| FSD | 功能规格 | 开发 |

## 四、PRD写作技巧

### 写作原则
1. **清晰明确**：避免模糊表述
2. **完整详尽**：覆盖所有场景
3. **逻辑一致**：前后对应
4. **可执行**：开发能直接执行

### 常见问题
- 需求描述不清晰
- 遗漏边界情况
- 优先级不明确
- 缺少验收标准

### 评审要点
- 需求完整性
- 逻辑一致性
- 技术可行性
- 测试可覆盖性
