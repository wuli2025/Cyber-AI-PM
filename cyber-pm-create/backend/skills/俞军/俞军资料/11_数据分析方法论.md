# 数据分析方法论

## 一、数据分析概述

### 为什么数据重要
- 量化产品效果
- 指导产品决策
- 验证产品假设
- 发现产品问题

### 数据分析流程
```
发现问题 → 提出假设 → 收集数据 → 分析数据 → 得出结论 → 指导行动
```

---

## 二、核心指标体系

### 1. 北极星指标

#### 定义
唯一关键指标，代表产品为用户创造的核心价值

#### 选择标准
- 反映用户价值
- 可衡量可追踪
- 团队可影响
- 先导指标（非滞后指标）

#### 案例
| 产品类型 | 北极星指标 |
|----------|-----------|
| 抖音 | 日均使用时长 |
| 微信 | 日活跃用户数 |
| 电商 | GMV |
| 会员产品 | 付费转化率 |

---

### 2. 指标分层

#### AARRR模型
- **Acquisition**：获取用户
- **Activation**：激活用户
- **Retention**：留存用户
- **Revenue**：增加收入
- **Referral**：推荐传播

#### 用户漏斗
```
访问 → 注册 → 首次使用 → 活跃 → 付费 → 推荐
```

---

### 3. 常用指标

#### 流量指标
| 指标 | 定义 | 用途 |
|------|------|------|
| PV | 页面浏览量 | 页面热度 |
| UV | 独立访客 | 用户规模 |
| 访问时长 | 用户停留时间 | 内容吸引度 |
| 跳出率 | 只访问一页离开的比例 | 落地页效果 |
| 人均页面数 | 平均每人浏览页面数 | 用户深度 |

#### 活跃指标
| 指标 | 定义 | 用途 |
|------|------|------|
| DAU | 日活跃用户数 | 短期活跃度 |
| WAU | 周活跃用户数 | 中期活跃度 |
| MAU | 月活跃用户数 | 整体规模 |
| DAU/MAU | 月活用户中每日活跃比例 | 用户粘性 |

#### 留存指标
| 指标 | 定义 | 计算方式 |
|------|------|----------|
| 次日留存 | 首次使用后次日回访 | 次日留存数/新增用户数 |
| 7日留存 | 首次使用后第7天回访 | 7日留存数/新增用户数 |
| 30日留存 | 首次使用后第30天回访 | 30日留存数/新增用户数 |

#### 转化指标
| 指标 | 定义 | 用途 |
|------|------|------|
| 转化率 | 完成目标的用户比例 | 流程效率 |
| 点击率(CTR) | 点击用户/曝光用户 | 广告效果 |
| 付费转化率 | 付费用户/活跃用户 | 变现能力 |

#### 收入指标
| 指标 | 定义 | 用途 |
|------|------|------|
| GMV | 成交总额 | 交易规模 |
| 收入 | 实际收入 | 变现能力 |
| ARPU | 平均每用户收入 | 用户价值 |
| ARPPU | 平均每付费用户收入 | 付费用户价值 |
| LTV | 用户生命周期价值 | 用户价值 |

## 三、数据分析方法

### 1. 对比分析

#### 时间对比
- 同比（今年 vs 去年）
- 环比（本期 vs 上期）
- 日对比（今日 vs 昨日）

#### 空间对比
- 产品线对比
- 渠道对比
- 地域对比

#### 群体对比
- 新老用户对比
- 高低价值用户对比
- 不同特征用户对比

---

### 2. 趋势分析

#### 用途
- 发现变化规律
- 预测未来趋势
- 识别异常波动

#### 常用图表
- 折线图
- 面积图
- 堆积图

---

### 3. 漏斗分析

#### 用途
- 分析转化路径
- 定位流失环节
- 优化转化效率

#### 步骤
1. 定义转化路径
2. 计算每步转化率
3. 识别关键流失点
4. 分析流失原因
5. 提出优化方案

---

### 4. 分布分析

#### 常用分布
- 用户活跃度分布
- 用户价值分布
- 业务数据分布

#### 分析维度
- 用户分群
- 行为分组
- 属性分层

---

### 5. 归因分析

#### 归因模型
- 首次触达：100%归因首次渠道
- 末次触达：100%归因最后一次渠道
- 线性归因：平均分配各触点
- 时间衰减：近期触点权重更高
- 位置归因：首尾触点权重更高

#### 应用场景
- 渠道效果评估
- 营销投放优化
- 用户路径分析

---

### 6. 留存分析

#### 留存曲线类型
- 下降型（自然衰减）
- L型（用户快速流失）
- 阶梯型（存在唤醒机制）
- 稳定型（高粘性产品）

#### 留存分析方法
- 自然留存 vs 引导留存
- 同期群分析
- 用户行为留存

---

### 7. 同期群分析

#### 用途
- 分析用户随时间的变化
- 评估产品改进效果
- 区分自然波动和产品影响

#### 操作步骤
1. 按时间划分用户群
2. 追踪各群后续行为
3. 对比不同群的表现
4. 分析变化原因

---

## 四、A/B测试

### 适用场景
- UI方案选择
- 功能开关
- 运营策略调整
- 价格测试

### 测试流程
```
提出假设 → 设计方案 → 确定样本 → 执行测试 → 收集数据 → 统计分析 → 得出结论
```

### 设计要点
- 单一变量
- 样本代表性
- 足够样本量
- 合理测试周期
- 避免干扰因素

### 统计分析
- 置信度：通常95%
- 统计显著性检验
- 最小可检测效果(MDE)

### 常见错误
- 样本选择偏差
- 测试时间不足
- 多重检验问题
- 忽略辛普森悖论

## 五、数据埋点

### 埋点类型

#### 客户端埋点
- 点击事件
- 页面曝光
- 停留时长
- 元素曝光

#### 服务端埋点
- 业务事件
- 订单数据
- API调用

### 埋点规范
- 事件命名统一
- 参数定义清晰
- 避免冗余
- 考虑扩展

### 埋点文档
```
事件名：page_view
触发时机：页面加载完成
携带参数：
  - page_name: 页面名称
  - referrer: 来源页面
  - user_id: 用户ID
```
